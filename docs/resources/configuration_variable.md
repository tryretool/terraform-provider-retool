# Resource: retool_configuration_variable

Manages a Retool configuration variable. Configuration variables are used to store environment-specific values that can be referenced throughout your Retool applications.

## Example Usage

```terraform
resource "retool_configuration_variable" "test_config_var" {
  name = "Test Config Var"
  values = [
    {
      environment_id = "a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d" # prod
      value          = "value1"
    },
    {
      environment_id = "b2c3d4e5-f6a7-4b8c-9d0e-1f2a3b4c5d6e" # staging
      value          = "value2"
    }
  ]
}

resource "retool_configuration_variable" "test_config_var_with_description" {
  name        = "Test Config Var with description"
  description = "This is a test configuration variable with a description"
  values = [
    {
      environment_id = "a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d" # prod
      value          = "value1"
    },
    {
      environment_id = "b2c3d4e5-f6a7-4b8c-9d0e-1f2a3b4c5d6e" # staging
      value          = "value2"
    }
  ]
}

resource "retool_configuration_variable" "test_config_var_as_secret" {
  name        = "Test Secret Config Var"
  description = "This is a secret configuration variable"
  secret      = true
  values = [
    {
      environment_id = "a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d" # prod
      value          = "secret_value1"
    },
    {
      environment_id = "b2c3d4e5-f6a7-4b8c-9d0e-1f2a3b4c5d6e" # staging
      value          = "secret_value2"
    }
  ]
}
```

## Importing

Configuration variables can be imported using their ID:

```bash
terraform import retool_configuration_variable.example a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d
```

### Importing Secret Configuration Variables

When importing a configuration variable with `secret = true`, the Retool API returns encrypted placeholder values instead of the actual secret values. After importing:

1. The `secret` field will be correctly set to `true`
2. The `values` field will contain `null` for all value fields
3. You **must** provide the actual secret values in your Terraform configuration file

**Example workflow for importing a secret:**

```bash
# 1. Import the resource
terraform import retool_configuration_variable.my_secret abc-123-def

# 2. Add the resource to your .tf file with the actual secret values
# (Terraform will show you the other fields from the import)
```

```terraform
resource "retool_configuration_variable" "my_secret" {
  name        = "My Secret Variable"  # From import
  description = "Description"         # From import
  secret      = true                  # From import
  
  values = [
    {
      environment_id = "env-id-1"     # From import
      value          = "actual_secret_value_1"  # You must provide this
    },
    {
      environment_id = "env-id-2"     # From import
      value          = "actual_secret_value_2"  # You must provide this
    }
  ]
}
```

```bash
# 3. Run terraform plan to verify
terraform plan  # Should show no changes if values match what's in Retool
```

**Note:** Because secret values cannot be retrieved from the Retool API, you must obtain the actual secret values from your team or secrets management system and add them to your Terraform configuration after import.
